O que é o Git

Git é um sistema de controle de versão distribuído (DVCS) usado para rastrear alterações no código-fonte durante o desenvolvimento de software. Foi criado por Linus Torvalds em 2005 para o desenvolvimento do kernel Linux.

O Git permite que múltiplos desenvolvedores trabalhem no mesmo projeto simultaneamente sem interferir no trabalho uns dos outros. Ele registra o histórico de alterações e permite reverter para versões anteriores quando necessário.

O que é o GitHub

GitHub é uma plataforma de hospedagem de código-fonte baseada em Git. Ele fornece um serviço web onde os desenvolvedores podem armazenar seus projetos, colaborar com outros desenvolvedores e contribuir para projetos de código aberto.

Além do controle de versão, o GitHub oferece recursos como issue tracking, pull requests, documentação wiki e integração com diversas ferramentas de desenvolvimento.

Diferenças entre Git e GitHub

  • Git é um software de controle de versão que você instala localmente na sua máquina.
  • GitHub é um serviço online que hospeda repositórios Git.
  • Git é a ferramenta, GitHub é a plataforma que utiliza essa ferramenta.
  • Com Git você pode trabalhar offline, enquanto o GitHub requer conexão com a internet para a maioria das operações.
  • Git é open source e gratuito, GitHub oferece planos gratuitos e pagos com funcionalidades adicionais.

Principais comandos Linux para usar o Git Bash

ls - Lista arquivos e diretórios

cd [diretório] - Muda para o diretório especificado

pwd - Mostra o caminho do diretório atual

mkdir [nome] - Cria um novo diretório

rm [arquivo] - Remove um arquivo

rm -r [diretório] - Remove um diretório e seu conteúdo

cp [origem] [destino] - Copia arquivos ou diretórios

mv [origem] [destino] - Move ou renomeia arquivos ou diretórios

cat [arquivo] - Exibe o conteúdo de um arquivo

Principais comandos Git presentes no readme.md do repositório aula_git

git init - Inicializa um repositório Git

git status - Mostra o estado atual do repositório

git add [arquivo] - Adiciona arquivos ao staging area

git commit -m "mensagem" - Cria um commit com as alterações

git log - Mostra o histórico de commits

git branch - Lista, cria ou deleta branches

git checkout [branch] - Muda para uma branch específica

git merge [branch] - Mescla uma branch com a branch atual

git remote add origin [url] - Adiciona um repositório remoto

git push -u origin main - Envia commits para o repositório remoto

git clone [url] - Clona um repositório remoto

Como criar um repositório no GitHub

Passo 1: Fazer login na sua conta GitHub

Acesse github.com e faça login na sua conta.

Passo 2: Clicar no botão "New"

No canto superior direito da página, clique no botão "+" e selecione "New repository".

Passo 3: Preencher as informações do repositório

Digite um nome para o repositório, uma descrição (opcional) e escolha se será público ou privado.

Passo 4: Inicializar com um README (opcional)

Marque a opção "Initialize this repository with a README" se quiser começar com um arquivo README.md.

Passo 5: Criar o repositório

Clique no botão "Create repository" para finalizar a criação.

Como ligar um projeto existente na máquina

Passo 1: Inicializar o Git no projeto

No terminal, navegue até a pasta do projeto e execute: git init

Passo 2: Adicionar o repositório remoto

Execute: git remote add origin [URL_DO_SEU_REPOSITORIO]

Passo 3: Verificar o remote

Execute: git remote -v para confirmar que o repositório remoto foi adicionado corretamente.

Como gerar uma versão com Git

Passo 1: Adicionar arquivos ao staging

Execute: git add . para adicionar todos os arquivos modificados, ou git add [arquivo] para arquivos específicos.

Passo 2: Fazer commit das alterações

Execute: git commit -m "Mensagem descritiva do commit"

Como enviar da máquina para o GitHub

Passo 1: Fazer commit das alterações

Certifique-se de que todas as alterações foram commitadas (veja a seção anterior).

Passo 2: Enviar para o repositório remoto

Execute: git push origin main (ou "master" dependendo do nome da sua branch principal)

Passo 3: Verificar no GitHub

Acesse seu repositório no GitHub para confirmar que as alterações foram enviadas com sucesso.
